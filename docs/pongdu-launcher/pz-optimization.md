# PZ 원클릭 최적화

특수좀비 소환, 강령술 등으로 좀비 개체수가 순간적으로 늘어나는 상황에 대비해, Project Zomboid의 좀비 연산 관련 클래스와 메모리 설정을 자동으로 패치하는 기능입니다.

!!! warning "B41 전용"

    이 기능은 Build 41 클래스 구조를 대상으로 합니다. 대상 파일이 하나라도 예상과 다르면(B42로 전환된 경우 등) **아무것도 건드리지 않고 중단**됩니다.

## 무엇을 바꾸나

| 대상 | 내용 |
| --- | --- |
| 좀비 연산 클래스 9종 | `zombie/iso`, `zombie/popman` 경로의 `IsoChunkMap`, `IsoWorld` 계열, `NetworkZombiePacker`, `ZombieCountOptimiser` 등 |
| 메모리 힙 | `ProjectZomboid64.json`의 `vmArgs`(-Xms/-Xmx)를 **전체 RAM의 절반**(256MB 단위 절사, 최소 2048MB)으로 설정 |
| 실행 배치파일 | `ProjectZomboid64.bat`도 동일하게 힙 값 치환 (그대로 두면 배치파일로 직접 실행할 때 기본값 3GB로 돌아가서) |

## 자동 탐지

Steam 설치를 기준으로 동작합니다.

1. 레지스트리(`HKCU\Software\Valve\Steam` 등)로 Steam 루트 탐색
2. 못 찾으면 `C:`~`H:` 드라이브의 `Steam` / `SteamLibrary` 폴더를 직접 스캔
3. 찾은 각 Steam 루트에서 `libraryfolders.vdf`를 파싱해 추가 라이브러리 폴더까지 전부 확인
4. `appmanifest_108600.acf`(PZ의 Steam AppID)가 있는 라이브러리를 우선 사용

## 적용 흐름

- **최초 실행**: 별도 동의창 없이 자동으로 적용되고, 이후에도 유지되도록 `opt_mode: "auto"`가 저장됩니다.
- **이후 실행**: 상태가 어긋나 있으면(게임 업데이트로 클래스가 되돌아갔거나 RAM 구성이 바뀐 경우) 조용히 다시 적용합니다.
- **사용자가 "해제"를 누른 경우**: `opt_mode: "off"`로 저장되어, 앱을 껐다 켜도 자동으로 다시 적용되지 않습니다.
- **PZ가 실행 중이면** 파일을 교체할 수 없으므로 적용을 건너뛰고 "게임 최적화 대기" 상태로 표시됩니다. 게임을 끄고 퐁듀를 재시작하세요.

## 백업과 복원

- 최초 적용 시 원본 파일이 게임 폴더 안 `puppet_opt_backup/`에 한 번만 백업됩니다(이미 백업이 있으면 덮어쓰지 않아 원본이 보존됩니다).
- **해제** 버튼을 누르면 이 백업에서 원본을 그대로 복원합니다.
- 백업 폴더 자체가 없다면(백업 전에 파일이 유실된 경우) 복원이 불가능하며, 이 경우 Steam의 "게임 파일 무결성 검사"로 복구해야 합니다.

## 상태 판정

메인 창(런처) 하단에 아래 세 가지 상태 중 하나가 표시됩니다.

| 상태 | 의미 |
| --- | --- |
| 적용됨 | 클래스 9종 + 힙 설정이 모두 일치 |
| 일부만 적용됨 | 클래스 일부만 패치된 상태 — 보통 게임 업데이트로 원본이 되돌아간 경우. 재적용 필요 |
| 미적용 | 패치되지 않은 원본 상태 |

## 관리자 권한이 필요한 경우

게임이 `Program Files` 등 일반 권한으로 쓸 수 없는 경로에 설치되어 있으면, 적용/해제 시 Windows UAC 승격 창이 뜹니다. 퐁듀 자기 자신을 `--pz-optimize` / `--pz-restore` 플래그로 관리자 권한 재실행하는 방식이며, 이 승격 프로세스는 퐁듀 본체를 종료하지 않고 별도로 동작합니다.
