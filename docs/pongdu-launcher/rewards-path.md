# rewards.txt 경로

퐁듀와 PZ 모드는 `rewards.txt` 파일 하나로 통신합니다. 퐁듀가 후원 이벤트를 이 파일에 한 줄씩 추가하면, 모드의 `rewardManager.lua`가 그 줄을 읽어 인게임 이벤트를 발동시킵니다.

## 자동 탐지

퐁듀는 실행할 때마다 아래 순서로 경로를 자동 탐지합니다.

1. `~/Zomboid/Lua/rewards.txt`
2. `%USERPROFILE%/Zomboid/Lua/rewards.txt`
3. OneDrive로 리다이렉트된 경우: `%OneDrive%/Zomboid/Lua/rewards.txt`
4. 위에서 못 찾으면 `C:`~`F:` 드라이브를 돌면서 `Users/*/Zomboid/Lua` 폴더를 탐색

대부분의 환경에서는 자동 탐지만으로 충분합니다.

## 수동 조작

메인 창 상단의 "rewards.txt 경로" 항목에서 두 가지를 조작할 수 있습니다.

| 버튼 | 동작 |
| --- | --- |
| 다시 탐지 | 위 자동 탐지 로직을 다시 실행 |
| 직접 지정 | 파일 탐색기로 원하는 경로를 직접 선택 |

경로를 직접 지정하면 `chzzk_donation_config.json`에 저장되어 다음 실행부터 자동 탐지 대신 그 경로를 그대로 사용합니다.

!!! warning "경로가 없으면 연동도, 테스트도 불가"

    rewards.txt 경로가 비어 있으면 연동 시작과 테스트 후원 주입 모두 막힙니다. "경로가 없어서 테스트 불가"라는 로그가 뜨면 경로부터 지정하세요.

## 파일 형식

한 줄에 후원 하나, 필드는 콤마로 구분된 4개입니다.

```
금액,featureId,후원자,메시지
```

- `featureId`가 비어 있으면(해당 금액에 매핑된 기능이 없으면) 모드 쪽에서는 통계 용도로만 기록되고 실제 인게임 효과는 발동하지 않습니다.
- 후원자명·메시지의 콤마, 줄바꿈, `%` 문자만 퍼센트 인코딩되고 한글을 포함한 나머지 유니코드는 원본 그대로 기록됩니다 (모드 쪽 Lua `urldecode`와 맞춰진 방식).
- 파일은 CRLF 줄바꿈으로 이어서 기록(append)됩니다. 폴더가 없으면 자동으로 생성합니다.
